# 核心知识

从经验中提炼的、已验证的核心知识。这是最重要的文件。

---

## 规则

- 无法快速获取的外部资源链接，直接告诉用户并提供替代方案，不要反复搜索
- 同一页面的链接应保持使用同一平台/来源
- Git推送前先检查代理环境变量
- 代理连接优先使用socks5协议

## 模式

- **GitHub Pages更新流程**：修改文件 → git add → git commit → git push
- **命令行代理设置**：`$env:HTTPS_PROXY="socks5://127.0.0.1:端口号"`

## 最佳实践

- 遇到网络问题时，先问用户代理端口和协议
- 用户说"你自己找"时，如果确实找不到，不要硬找，直接说明情况
